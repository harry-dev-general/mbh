Example Queries for Linked Records:

Find bookings where you're the onboarding employee:

AND({Status}='PAID',SEARCH('YOUR_EMPLOYEE_ID',{Onboarding Employee}&''))

Find bookings where you're the deloading employee:

AND({Status}='PAID',SEARCH('YOUR_EMPLOYEE_ID',{Deloading Employee}&''))

Why concatenate with empty string? Linked record fields in Airtable are arrays. The SEARCH() function needs a string to search in, so we concatenate the field with an empty string (&'') to convert it to a searchable format.
